# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Balans** is a single-page Dutch household finance app for two partners to manage shared expenses. It parses Rabobank PDF bank statements, matches transactions to fixed expenses, and tracks split contributions.

## Running the App

No build step — the app is a single self-contained `index.html` file.

```bash
# Local development: open directly in browser
open index.html

# Docker (serves on http://localhost:8080)
docker-compose up

# Syntax-check a JS block (no test suite exists)
node -e "const fs=require('fs'); new Function(fs.readFileSync('index.html','utf8')); console.log('OK')"
```

## Architecture

**Everything lives in `index.html`** (~2,800 lines of HTML + embedded CSS + vanilla JS). There are no separate source files, no npm packages, no build tools.

Key architectural decisions:
- **State** is a single object `S` persisted to `localStorage` under the key `"balans"` — all reads/writes go through `load()`/`persist()`.
- **Rendering** is imperative: each page has a dedicated `render*()` function that sets `innerHTML`. No virtual DOM.
- **Financial month** runs from the 24th of one month to the 23rd of the next (`MONTH_START_DAY = 24`). `financialMonthFromDate()` converts a calendar date to `{ y, m }`. `monthKey()` returns `"YYYY-MM"` for the active month.
- **PDF parsing** uses PDF.js (CDN, v3.11.174). `pickStatementParser(doc)` selects from `statementParsers[]` by calling each parser's `canParse(doc)`. The CC parser must be registered **before** the Rabobank Standard parser to avoid false-positive detection.

### State shape (`S`)

```
S = {
  y, m,  // active financial year/month
  settings: {
    p1, p2,          // partner names
    split,           // p1's share percentage (0–100)
    ownIbans: [],    // legacy, migrated to trackedIbans on load
    cars:     [ { id, name, licensePlate } ],
    salaryIbans:  [ { id, label, iban, ignore, minAmount, maxAmount } ],
    trackedIbans: [ { id, name, iban, ignore, pairCancel } ]
  },
  salaries: {
    [monthKey]: { p1, p2, streef1, creditcard1 }
    // streef1 = target salary for p1; creditcard1 = cc amount deducted from p1 transfer
  },
  expenses: [ {
    id, name, amount, months,         // months: 'all' or sorted number[] (1–12)
    categoryId, accountId,
    counterpartyIban, carId,
    tags, shared,
    weeklyInterval,                   // null = monthly; number = every N weeks
    weekday,                          // 0–6 matching JS getDay() (0=Sun); null when not weekly
    weekStartDate,                    // ISO date string; enables exact week counting
    dayOfMonth,                       // 1–31; used to score tx date proximity for matching
    contributors: [ { id, name, amount, iban, accountId, paymentAmount } ]
    // paymentAmount: contributor's TOTAL monthly transfer (may cover multiple expenses)
  } ],
  categories: [ { id, name } ],
  accounts:   [ { id, name } ],
  imports: [ {
    id, accountId, statementAccountIban, fileName,
    periodStart, periodEnd, parserId, parserLabel,
    transactions: [ {
      id, bookedDate, amount, direction,
      typeCode, counterpartyIban, counterparty,
      description, accountId,
      matchedExpenseId, matchScore, matchReason,
      manualExpenseId,   // set when user manually links; survives recompute
      dismissed,
      splits,            // [{id, expenseId, amount, contributorName}] — manual or auto
      autoSplit          // true = generated by recompute; false = manual (never overwritten)
    } ]
  } ]
}
```

### UI Sections (tabs)

- **Maand** — salary inputs (p1: salaris + streefsalaris + creditcard; p2: salaris), expense list with filter pills (Alles/Verwerkt/Onverwerkt), income/expense summary, transfer amounts per partner + Autorekening.
- **Uitgaven** — CRUD for fixed recurring expenses. Account filter pills shown when expenses span multiple accounts. Shows contributor lines with "verwerkt" badge.
- **Transacties** — import PDF statements; shows **all** unmatched transactions (not filtered by active month). Account filter pills + type filter (Alles/Afschrijvingen/Bijschrijvingen). Credit transactions can be linked to one expense (Koppelen) or split across multiple expenses (Splitsen).
- **Instellingen** — partner names, split ratio, accounts, categories, salarisrekeningen, andere rekeningen (trackedIbans), auto's.

### PDF parsers (`statementParsers[]`)

Each parser: `{ id, label, canParse(doc), parse(doc) }`. Registered in order — first match wins:

1. **`rabo-cc-v1`** (`parseRaboCCStatement`) — Rabobank GoldCard Mastercard. Detects via `GoldCard Mastercard` on page 1. IBAN has `EUR` suffix in source; positive amounts = credit (payments to card), negative = debit (purchases). Amounts in European format with `.` thousands separator.
2. **`rabo-standard-v1`** (`parseRaboStandardStatement`) — Rabobank Standaard rekeningoverzicht.

### Transaction matching

`recomputeAllImportMatches()` runs at boot and after every expense/import change. Priority order per transaction:

1. **`tx.manualExpenseId` set** → restore single-link, never overwrite.
2. **`(tx.splits.length || tx.dismissed) && !tx.autoSplit`** → manual split/dismiss, preserve as-is.
3. **Credit + counterpartyIban** → try `computeAutoSplitsForCredit(tx)`: if multiple expenses share this contributor IBAN and at least one has `contributor.paymentAmount` matching `tx.amount`, auto-split across all matching expenses. Sets `tx.autoSplit = true`; re-evaluated on next recompute.
4. **Otherwise** → `matchTransactionToExpense(tx)`: clears any stale auto-split state. For **debit** results, a `claimedDebitKeys` set (keyed `"expenseId|YYYY-MM"`) ensures only one debit transaction per expense per financial month is auto-linked — subsequent matches leave the transaction unmatched. Credits are never claimed this way (multiple contributors may credit the same expense).

**`matchTransactionToExpense` logic:**
- **Credits with IBAN**: matched to the first shared expense whose `contributors` array contains this IBAN (optionally filtered by `accountId`).
- **Debits**: candidates filtered by account + amount + active month, scored by `scoreExpenseMatch()`. Single candidate auto-links if score ≥ 18 or unique amount+account; multiple candidates require winner ≥ 24 AND gap ≥ 12 over second place.

**`scoreExpenseMatch` scoring:**
- Counterparty IBAN exact match: +100
- Full name match in tx text: +80
- Per matching token (name + tags): +18 each
- Category name match: +12
- License plate match (ANWB IBAN): +100
- `dayOfMonth` proximity: +60 (exact), +35 (≤2 days off), +15 (≤5 days off) — critical for disambiguating same-amount same-name expenses on CC statements

### Transaction splits

**Manual splits** (via "Splitsen" button on Transacties page):
- Draft state stored in `splitDrafts` (in-memory, keyed by txId).
- `saveSplits(txId)` writes `tx.splits`, sets `tx.dismissed = true`, `tx.autoSplit = false`.
- After `renderTransactions()` rebuilds the list, `renderSplitFormFor(txId)` re-populates any open draft forms surgically (avoids focus loss on input).

**Auto-splits** (from contributor `paymentAmount`):
- Configure on expense: set contributor IBAN + their share `amount` + `paymentAmount` (total they transfer).
- When a credit arrives matching `paymentAmount` and multiple expenses share the IBAN → auto-splits created with each expense's `amount`. Marked `tx.autoSplit = true` so recompute can refresh them if contributor data changes.

Both manual and auto splits feed into `matchedCreditTxsByExpenseForActiveMonth()`, which returns `{ expenseId: [{counterparty, iban, amount}] }` used everywhere contributor lines are rendered.

### Auto-dismiss (`autoVerwerktIds`)

Computed dynamically (never persisted). Suppresses transaction IDs from the unmatched list:
1. **pairCancel** (`trackedIban.pairCancel = true`): same-day same-amount debit+credit pairs from that IBAN.
2. **salaryIbans** (`ignore = true`): credits from that IBAN within optional `minAmount`/`maxAmount`.
3. **trackedIbans** (`ignore = true`): any transaction from that IBAN.
4. **Autorekening** (`NL17INGB0112314074`): debit matching previous financial month's `p1sal − streef1`.
5. **CC-specific** (imports with `parserId === 'rabo-cc-v1'`):
   - Same-day same-amount debit+credit pairs (no IBAN required).
   - Entries matching `/verrekening vorig overzicht/i`.

### Weekly expenses

`effectiveExpenseAmount(e, y, m)` returns `e.amount × weeklyOccurrencesInMonth(weekday, interval, y, m, weekStartDate)`.

`weeklyOccurrencesInMonth` uses noon timestamps (DST-safe):
- **With `weekStartDate`**: exact count — advances from `startDate` in `interval`-week steps, counts hits within the financial month's date range.
- **Without `weekStartDate`**: counts weekday occurrences in range, divides by interval, rounds.

### Account badge colours

`ACCOUNT_COLORS` is a palette of 5 colour pairs (blue, orange, green, purple, rose). `accountColorStyle(accountId)` picks by the account's index in `S.accounts`. `accountBadgeHtml(accountId)` emits an inline-styled `<span class="badge">`. The weekly occurrence badge (`N×`) still uses the static `.badge-account` class (always blue).

### Maand calculation

- `p1calc = streef1 || p1sal` — streefsalaris used for income when set; falls back to actual salary.
- `gross` = sum of `effectiveExpenseAmount(e, S.y, S.m)` across all month expenses.
- `contribs` = static contributor amounts + dynamic amounts from credit txs (including splits) linked to expenses in the active month.
- Transfer amounts: `t1 = over * split - creditcard1`, `t2 = over * (1 - split)`, `autorekening = p1sal - streef1`.

### IBAN normalisation

`normalizeIban(value)` strips spaces and uppercases. Always use this before comparing or storing IBANs. NL IBANs are 18 characters (`NL` + 2 digits + 4 letters + 10 digits).
